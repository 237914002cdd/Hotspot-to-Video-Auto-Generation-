## HyperFrames 脚本生成规范

### 一、HTML 结构

**根元素**：`data-composition-id` + `data-start` + `data-duration` + `data-width`/`data-height`

```html
<div id="root" data-composition-id="main" data-start="0" data-duration="31.7"
     data-width="1080" data-height="1920">
```

**场景定义**：每个场景用 `<div class="clip">` 包裹，通过 `data-start` 和 `data-duration` 切分时间轴。

```html
<!-- 场景 1：0-5.45s -->
<div class="clip" data-start="0" data-duration="5.45" data-track-index="1">
  <div class="sn">01 / 04</div>
  <div id="s1-title" class="h1" style="...">标题</div>
</div>

<!-- 场景 2：5.45-18.22s -->
<div class="clip" data-start="5.45" data-duration="12.77" data-track-index="1">
  ...
</div>
```

**属性说明**：
| 属性 | 说明 |
|------|------|
| `data-start` | 起始时间（秒），如 `"0"`、`"5.45"` |
| `data-duration` | 持续时长（秒），如 `"5.45"` |
| `data-track-index` | 轨道层级（0=背景，1=前景内容，2=字幕） |
| `data-composition-id` | 合成唯一标识，根元素必需 |

**多轨道分离**：
- `track-index=0`：渐变/光晕背景
- `track-index=1`：主要内容（标题、卡片、标签）
- `track-index=2`：字幕（独立叠加，永远在最上层）
- `track-index=3`：音频

### 二、GSAP 动效规范（审美基准）

#### 2.1 大标题入场（禁止单纯 y 轴位移）

**差劲写法（禁止）：**
```javascript
tl.fromTo("#title", {opacity:0, y:45}, {opacity:1, y:0, duration:0.5});
```

**规范写法（必须）：**
```javascript
tl.fromTo("#title", {opacity:0, y:60, scale:0.95, skewX:-4},
  {opacity:1, y:0, scale:1, skewX:0, duration:0.7, ease:"back.out(1.2)"});
```

组合拳要素：`opacity` + `y` + `scale` + `skewX（可选）` + `ease:"back.out(1.x)"`

#### 2.2 光晕呼吸动画

每个场景的背景光晕不能只在开头闪一下，必须持续呼吸：

```javascript
tl.fromTo("#sX-glow", {scale:0.8, opacity:0.3},
  {scale:1.3, opacity:0.6, duration:5, ease:"power1.out"}, sceneStartTime);
```

#### 2.3 交错入场（stagger）

并列元素用 `stagger` 实现多米诺骨牌效果，不要单个写：

```javascript
// ✅ 规范：三行搞定
tl.fromTo(["#item1", "#item2", "#item3"], {opacity:0, x:-40},
  {opacity:1, x:0, stagger:0.15, ease:"power3.out"}, offsetTime);

// ❌ 差劲：逐个写三遍
tl.fromTo("#item1", ..., 6.9);
tl.fromTo("#item2", ..., 7.3);
tl.fromTo("#item3", ..., 7.7);
```

#### 2.4 CSS 过渡动画

部分效果（进度条、脉冲光晕、旋转）用 CSS transition 配合 `tl.call()` 更高效：

```javascript
tl.call(function(){
  var e = document.getElementById('bar');
  if(e) { e.style.transition = 'width 2.5s ease-out'; e.style.width = '300px'; }
}, [], offsetTime);

tl.call(function(){
  var e = document.getElementById('glow');
  if(e) { e.style.transition = 'box-shadow 1s ease-in-out infinite alternate';
          e.style.boxShadow = '0 0 40px rgba(255,159,10,0.3)'; }
}, [], offsetTime);
```

### 三、空间排版规范

#### 3.1 禁止全宽方块堆叠

**差劲（卡片全宽靠叠）：**
```
top:30% left:10% right:10%  ← 卡片1
top:44% left:10% right:10%  ← 卡片2
top:58% left:10% right:10%  ← 卡片3
```

**规范（错落有致）：**
- 并列 3 个特征用浮/网格排版：`left:8%` / `left:50%`(居中) / `right:8%`
- 或使用 display:flex + gap 让元素自然分布

#### 3.2 视觉中心原则

- 主标题：top 14-20%
- 核心卡片/内容：top 28-35%
- 辅助信息：top 48-55%
- 底部行动点：bottom 22-28%
- 所有文字居中：`text-align:center`

### 四、时间轴编排规范

```javascript
window.__timelines = window.__timelines || {};
var tl = gsap.timeline({ paused: true });

// 字幕数组
var subs = ['第1句', '第2句', '第3句'];

// S1: 0-5s
tl.fromTo("#s1-title", ..., 0.5);
tl.call(function(){subText.textContent=subs[0];subBar.style.opacity='1';},[],0.3);
tl.to("#subBar",{opacity:0,duration:0.3}, 4.7);

// S1 → S2 过渡
tl.to(".clip[data-start='0']", {opacity:0,duration:0.3}, 5.0);
tl.to(".clip[data-start='5.45']", {opacity:1,duration:0.4}, 5.45);

// S2: 5-12s
// ... 同理

window.__timelines["main"] = tl;
```

**关键规则：**
1. `paused: true` 必须有
2. 每个 `tl.call()` 中的 DOM 操作优先用 `style.transition` + 直接设置属性
3. 字幕在场景切换前先 fade out（`tl.to("#subBar",{opacity:0})`）
4. 场景切换用 `.clip[data-start='X']` 作为选择器
5. `window.__timelines["main"]` 注册必须在脚本末尾

## 上下文驱动生成系统（Phase 5 / 6.2 — 关键核心资产）

> 以下逻辑全部实现在 `server.js` 中，**严禁任何硬编码假文案**。

### 1. 封面内容注入（`parsePlanForCover()`）

**输入**：`01-内容方案/video-plan.md`（优先）→ 回退到 `short-video-copy.md`

**12 字段提取规则**：

| 字段 | 来源 |
|------|------|
| `badge` | 正文含 `2026` → `2026 最新`；含 `AI` → `AI 工具`；否则 `热门推荐` |
| `titleLine1/2` | `#` 标题行，按字符数 50% 分割为两行（上限 12 字/行） |
| `subtitle` | 首个非标题、长度 < 60 的正文行 |
| `feature1/2/3` | 先提取 `##` 子标题（去编号、去时间标注）；不足时用正文句填补 |
| `tag1Val` | 文中第一个数字 + `+` 后缀（如 `5000` → `5000+`） |
| `tag2Val` | 含 `教程` → `教程`；含 `AI` → `AI`；含 `签证` → `攻略`；否则 `指南` |
| `tag3Val` | 固定 `NEW` |
| `cta` / `footer` | 固定文案 |

**红线规则**：三个 feature 中任何字段若含 `#` 字符，视为提取不干净，自动降级为正文句填充。

### 2. 社交文案生成（`POST /api/projects/generate-blueprint`）

**动态提取流水线**：

```
video-plan.md
     ↓
提取真实内容行（去 # / > / |—— 格式行）
     ↓
检查表格格式 → 提取 | 分隔的 cols[2]（口播文案列）
     ↓
cols[2] ≥ 3 条 → 用表格口播列
      不足     → 用非表格正文句
     ↓
取前 5 个 → 格式清理（去 * 号、去 | 残余、去编号前缀）
     ↓
→ f1/f2/f3 注入双平台文案
```

**表格 vs 非表格矩阵**：

| 文件格式 | 提取路径 | 例 |
|---------|---------|-----|
| `\`步骤\|时长\|口播文案\`` 表格 | `cols[2]` | whv-2026 提取 WHV 签证步骤 |
| 纯段落正文 | `realSentences` 过滤 | apple-ai-wwdc26 提取 AI 功能点 |

**标签自动识别**：
- 含 `2026` / `WHV` → `#2026最新`
- 含 `AI` → `#AI工具`
- 否则 → `#推荐`

### 3. 模板注入架构

```
template.svg（带 {{PLACEHOLDER}} 占位符）
     ↓
parsePlanForCover() → 12 字段 data
     ↓
replaceAll 依次替换 15 个占位符
     ↓
→ 02-制作文件/cover.svg（编辑版）
→ 03-成品/cover.svg（正式版，归档用）
```

## 黄金规则（踩坑记录 — 每次复盘后更新）

### 1. 永远不要用硬编码文案！

**事故**：Phase 3.2 的 `generate-blueprint` 返回的 `copy` 对象中残留了"上手简单，零门槛"等 6 处万能文案，即使后端已从 `video-plan.md` 动态生成 HTML，文案部分仍在输出假内容。

**原因**：重构时只替换了 HTML 生成部分，遗漏了下面的 `copy` 对象。

**根治**：`generate-blueprint` 返回的所有文本必须追根溯源到 `video-plan.md`。每次改这个函数后，curl 检测调用一次：
```bash
curl -s http://localhost:3456/api/projects/generate-blueprint \
  -H 'Content-Type: application/json' \
  -d '{"slug":"whv-2026"}' | jq '.copy.xiaohongshu.body'
```

### 2. 旧进程残留导致新路由失效

**症状**：明明改了 `server.js` 的代码，重启后 API 返回 `404 Cannot POST`。

**诊断**：
```bash
# 查看旧 PID
netstat -ano | grep :3456
# 对比代码是否已载入
grep -n "apiRouter" server.js | tail
```

**根治**：先 kill 旧进程再启动，不要只按 Ctrl+C：
```bash
netstat -ano | grep :3456 | awk '{print $5}' | xargs taskkill //F //PID
```

### 3. 双轨蓝键转移逻辑

**优先级**：缺剧本 → 缺视频 → 缺封面 → 全部完成全灰。

`server.js` 中 `scanProjects()` 的 `hasScript` 检测必须**同时扫描根目录和 `hyperframes/` 子目录**。旧项目（如 `hyperframes-intro`）的 index.html 在 `hyperframes/` 下，漏扫会导致蓝键错位。

### 4. Windows open-folder 路径编码

**已试 5 种方案均失败**（exec/execFile/bat/mshta），最终方案：
```javascript
require('child_process').exec('cmd.exe /c start "" "' + target + '"', ...)
```
中文/空格路径必须用 `cmd.exe /c start ""` 包装。

### 5. `scanProjects()` 中的变量名冲突

- `const status = []` 与状态字符串 `status` 冲突 → 改用 `statusList`
- `const cPath` 重复声明 → 第二次重命名
- `const topic` 与上层参数 `topic` 冲突 → 删除多余声明
