# 产品愿景 — 视频操作系统

> 核心观点：别人做"剪辑软件"，我们做"视频操作系统"。
> 更新：2026-06-13

---

## 产品哲学

### AI 处理低价值重复劳动，人保留审美判决

当前市面上所有的视频工具都犯了一个根本错误：**它们把"降低门槛"当作目标，把"模板化"当作解决方案。**

结果是：
- 新手用模板做出千篇一律的内容，毫无差异化
- 高手想高度自定义，却被迫在时间轴上一帧帧调整

我们相信，正确的分工应该是：

```
AI 做的事情          → 人做的事情
────────────────────────────────────
搜罗热点并评分       → 决定"做不做这个选题"
写文案初稿            → 修改语气、调整节奏、确认是否要发
生成 HTML + GSAP 动效 → 在浏览器逐页审查效果
渲染 mp4（等 30 分钟）→ 播放确认是否有问题
出封面设计稿          → 一轮轮改到满意
搬运文件、写 README   → 点击一键归档触发
```

**AI 不取代人，AI 取代的是"重复劳动"**——热点调研、格式转换、文件搬运、多平台适配这些事不应该占用创作者的时间。

我们称之为 **三层分工架构**：

| 层 | 解决 | 主体 | 工作量占比 |
|----|------|------|-----------|
| AI 决策层 | 做什么 | 算法 | ~30% |
| SOP 执行层 | 怎么做 | 自动化流水线 | ~40% |
| **人工审美层** | **好不好** | **你** | **~30%（但决定质量）** |

## 问题定义

### 为什么需要"视频操作系统"而不是"模板生成器"？

现有工具的分类：

| 工具类型 | 代表 | 本质 | 天花板 |
|---------|------|------|--------|
| 剪辑软件 | CapCut / PR / Final Cut | 时间轴编辑器 | 每视频从零拖拽 |
| 代码视频框架 | Remotion / HyperFrames | 代码→视频编译器 | 有代码无流水线 |
| AI 批量生成 | 剪映 AI 成片 / InVideo | 文案→视频映射器 | 审美不可控 |

**它们共同的盲区**：以上所有工具都只解决了"怎么做视频"这一步。但创作者面临的真正问题是：

> **我现在应该做什么视频？**
> **做完这个视频后，下一个做什么？**
> **哪个类型的视频效果最好？**

这不是剪辑问题，这是**运营问题**。

我们的方案不是另一个"做视频的工具"，而是一套 **视频操作系统**——它接管了从"发现热点"到"归档复盘"的全链路。

类比：
```
Windows 管理你的文件  → 文件系统 + 资源管理器
macOS 管理你的应用    → 应用商店 + Launchpad
本系统管理你的视频生产 → 看板 + SOP 流水线 + 资产追踪
```

## 未来路线图

> 以下按成熟度排序。已实现的标注 ✅，规划中的标注 ⏳。

### 阶段一：核心流水线 ✅（当前版本）

```
热点发现 → 选题评分 → 自动建项目 → 剧本生成 → 封面注入
→ 渲染 → 双平台文案 → 一键归档 → Obsidian 同步
```

**已完工**：10 个 Phase，15 个 API 端点，6 项目验证。

### 阶段二：增长反馈闭环 ⏳（**下一步**）

```
┌─────────────────────────────────────┐
│  发布追踪                           │
│  ├── 项目归档后标记"已发布/待发布"    │
│  ├── 记录各平台发布链接              │
│  └── 首次播放数据手动录入            │
├─────────────────────────────────────┤
│  数据看板                           │
│  ├── 播放量 vs 选题评分对比          │
│  ├── 哪个平台转化率高                │
│  └── 哪个话题类型最受欢迎            │
├─────────────────────────────────────┤
│  权重反哺                           │
│  └── 播放数据 → 自动调整选题评分权重  │
└─────────────────────────────────────┘
```

### 阶段三：多平台自动发布 ⏳

- 对接小红书开放平台 API（自动发布）
- 对接抖音开放平台 API（自动发布）
- 定时排期（周一发小红书、周三发抖音）
- 评论互动数据回传

### 阶段四：团队协作 ⏳

- 多角色权限（选题人 → 制作人 → 审核人 → 发布人）
- 看板插件化（HyperFrames 动效模板市场）
- 资产库（跨项目复用封面模板、GSAP 动效片段）

### 阶段五：开放生态 ⏳

- API 对外开放（已有 15 个端点）
- 自定义 SOP 插件
- HyperFrames 动效模板市场

> **我们解决"做什么、怎么做、好不好、然后呢"——从头到尾的生产管理。**
>
> **这是剪辑软件到视频操作系统的进化。**
